8. Prototipos, Herencia

8.1 Herencia Prototípica

Este apartado fundamental en JavaScript explora cómo los objetos pueden heredar propiedades y métodos de otros objetos. A diferencia de la herencia basada en clases de otros lenguajes, JavaScript utiliza un mecanismo de "cadena de prototipos". Cuando se intenta acceder a una propiedad de un objeto, si esta no se encuentra directamente en él, JavaScript busca la propiedad en el prototipo de ese objeto, y luego en el prototipo del prototipo, y así sucesivamente, hasta llegar a null.

8.2 Prototipo F

Cuando trabajamos con funciones constructoras (por ejemplo, function Persona() {...}), cada función en JavaScript tiene una propiedad especial llamada prototype (F.prototype). Esta propiedad es un objeto y es donde se definen los métodos que serán heredados por todas las instancias creadas con esa función constructora usando la palabra clave new. Esto permite que todas las instancias compartan los mismos métodos, ahorrando memoria y fomentando la herencia.

8.3 Prototipos Nativos

JavaScript viene con una serie de objetos y constructores "nativos" (incorporados) como Object, Array, Function, String, Number, Date, etc. Cada uno de estos tiene su propio prototipo (ej., Array.prototype, String.prototype) que contiene los métodos y propiedades por defecto que usan sus instancias (ej., .push() para arrays, .toUpperCase() para strings). Es posible extender estos prototipos nativos, aunque debe hacerse con extrema precaución para evitar conflictos.
